# Critérios do Beta Program — SignallQ

> Atualizado em 2026-06-28.

## Closed Beta (M2 — 31/07/2026)

### Critérios de Entrada

- App com todas as features MVP ativas e testadas
- Build release assinado e distribuído via Firebase App Distribution
- Política de privacidade publicada
- Formulário de feedback disponível (Google Forms ou in-app)
- Mínimo 10 testers selecionados e convidados

### Perfil dos Testers (mín. 10, ideal 20-30)

| Perfil | Quantidade | Objetivo |
|---|---|---|
| Power users (técnicos) | 3-5 | Encontrar bugs, feedback detalhado |
| Usuários comuns | 3-5 | Validar usabilidade |
| Dispositivos antigos (API 24-28) | 2-3 | Compatibilidade |
| Dispositivos topo de linha | 2-3 | Performance baseline |
| Operadoras variadas | Distribuído | Cobrir Claro, Vivo, Tim, Oi |

### Critérios de Saída (para avançar ao M3)

- [ ] >= 14 dias de beta ativo
- [ ] >= 10 testers com pelo menos 3 sessões cada
- [ ] Zero bugs P0 abertos
- [ ] Bugs P1 <= 2 (com fix planejado)
- [ ] Crash-free rate >= 99%
- [ ] Feedback processado e triado
- [ ] Speedtest, diagnóstico IA e histórico validados por testers

---

## Open Beta (M4 — 21/08/2026)

### Critérios de Entrada

- Closed Beta concluído com critérios de saída atendidos
- App publicado na Play Store (track de teste aberto)
- Landing page do SignallQ no ar
- Email de suporte configurado
- FAQ publicado

### Critérios de Saída (para avançar ao M5)

- [ ] >= 14 dias de open beta
- [ ] >= 100 instalações
- [ ] Crash-free rate >= 99.5%
- [ ] ANR rate < 0.47%
- [ ] Rating médio >= 4.0 (se houver reviews)
- [ ] Nenhum feedback recorrente não endereçado
- [ ] Plano de hypercare ativo

---

## Formulário de Feedback

Campos obrigatórios:
1. Dispositivo e versão Android
2. Operadora
3. O que estava fazendo quando encontrou o problema
4. Descrição do problema ou sugestão
5. Screenshot (opcional)

Distribuição: link no app (tela de Ajustes) + email para testers.
